## Overview

Theme extensions customize Zed's appearance by defining color schemes for the editor, UI elements, syntax highlighting, and terminal. Themes are defined in JSON files that adhere to Zed's theme schema.

## Creating a Theme Extension

### Directory Structure

```
my-theme-extension/
  extension.toml
  themes/
    my-theme.json
    my-other-theme.json
```

### Extension Manifest

```toml
id = "my-theme-theme"
name = "My Theme"
version = "0.0.1"
schema_version = 1
authors = ["Your Name <you@example.com>"]
description = "Beautiful color scheme for Zed"
repository = "https://github.com/your-name/my-theme-zed"

themes = ["themes/my-theme.json"]
```

<Warning>
Theme extension IDs must be suffixed with `-theme` (e.g., `dracula-theme`, `nord-theme`) for publication.
</Warning>

## Theme JSON Structure

Themes use the [Zed Theme JSON Schema v0.2.0](https://zed.dev/schema/themes/v0.2.0.json).

### Theme Family

A theme file defines a theme family containing one or more themes:

```json
{
  "$schema": "https://zed.dev/schema/themes/v0.2.0.json",
  "name": "My Theme",
  "author": "Your Name",
  "themes": [
    {
      "name": "My Theme Dark",
      "appearance": "dark",
      "style": {
        // Theme colors
      }
    },
    {
      "name": "My Theme Light",
      "appearance": "light",
      "style": {
        // Theme colors
      }
    }
  ]
}
```

**Theme Family Fields:**

- `name` - Family name
- `author` - Theme author
- `themes` - Array of theme objects

**Theme Object Fields:**

- `name` - Individual theme name
- `appearance` - Either `"dark"` or `"light"`
- `style` - Color definitions

## Theme Style Properties

### Core Colors

```json
{
  "style": {
    "background": "#1e1e2e",
    "foreground": "#cdd6f4",
    "accent": "#89b4fa",
    "border": "#313244",
    "border.variant": "#45475a",
    "border.focused": "#89b4fa",
    "border.selected": "#94e2d5",
    "border.transparent": "#00000000",
    "border.disabled": "#6c7086"
  }
}
```

**Essential Properties:**

- `background` - Main background color
- `foreground` - Primary text color
- `accent` - Accent color for highlights and emphasis
- `border` - Default border color

## Theme Builder

Use [Zed's Theme Builder](https://zed.dev/theme-builder) to design themes visually:

1. Start with an existing theme
2. Adjust colors using the interactive editor
3. Preview changes in real-time
4. Export the JSON for your extension

The Theme Builder provides fine-grained control over all theme surfaces and shows live previews of editor UI, syntax highlighting, and terminal colors.

## Publishing Theme Extensions

Follow the standard [extension publishing process](/extensions/developing#publishing-extensions):

1. Add your theme repository as a submodule to [zed-industries/extensions](https://github.com/zed-industries/extensions)
2. Update `extensions.toml` with your theme metadata
3. Ensure your extension ID ends with `-theme`
4. Open a pull request

## Theme Development Tips

<AccordionGroup>
  <Accordion title="Test with Multiple Languages">
    Test your theme with various programming languages to ensure syntax highlighting works well across different code patterns.
  </Accordion>
  
  <Accordion title="Consider Accessibility">
    Ensure sufficient contrast between foreground and background colors. Use tools like [WebAIM Contrast Checker](https://webaim.org/resources/contrastchecker/) to verify readability.
  </Accordion>
  
  <Accordion title="Provide Light and Dark Variants">
    Many users switch between light and dark modes. Offering both variants in your theme family improves user experience.
  </Accordion>
  
  <Accordion title="Use Semantic Color Names">
    Choose descriptive color variable names in your design system before mapping them to theme properties.
  </Accordion>
</AccordionGroup>
