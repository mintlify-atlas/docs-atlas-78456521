## Overview

Elements are the workhorses of GPUI. They are responsible for laying out and painting all contents of a window. Elements form a tree and are laid out according to web layout standards as implemented by [Taffy](https://github.com/DioxusLabs/taffy).

## Element Basics

Elements are constructed by calling `Render::render()` on the root view of the window, which recursively constructs the element tree from the current state of the application.

<Steps>
  <Step title="Construction">
    `Render::render()` is called on the root view to build the element tree
  </Step>
  <Step title="Layout">
    Elements are laid out by Taffy using flexbox rules
  </Step>
  <Step title="Paint">
    Elements paint to the screen via their `Element::paint()` implementation
  </Step>
  <Step title="Cleanup">
    The entire element tree and callbacks are dropped before the next frame
  </Step>
</Steps>

## The Element Trait

The `Element` trait is the low-level, imperative API to GPUI.

```rust Element trait (simplified)
pub trait Element: 'static + IntoElement {
    type RequestLayoutState: 'static;
    type PrepaintState: 'static;

    fn id(&self) -> Option<ElementId>;
    
    fn request_layout(
        &mut self,
        id: Option<&GlobalElementId>,
        inspector_id: Option<&InspectorElementId>,
        window: &mut Window,
        cx: &mut App,
    ) -> (LayoutId, Self::RequestLayoutState);

    fn prepaint(
        &mut self,
        id: Option<&GlobalElementId>,
        inspector_id: Option<&InspectorElementId>,
        bounds: Bounds<Pixels>,
        request_layout: &mut Self::RequestLayoutState,
        window: &mut Window,
        cx: &mut App,
    ) -> Self::PrepaintState;

    fn paint(
        &mut self,
        id: Option<&GlobalElementId>,
        inspector_id: Option<&InspectorElementId>,
        bounds: Bounds<Pixels>,
        request_layout: &mut Self::RequestLayoutState,
        prepaint: &mut Self::PrepaintState,
        window: &mut Window,
        cx: &mut App,
    );
}
```

## IntoElement Trait

The `IntoElement` trait allows types to be automatically converted into elements.

```rust IntoElement trait
pub trait IntoElement: Sized {
    type Element: Element;

    fn into_element(self) -> Self::Element;
    fn into_any_element(self) -> AnyElement {
        self.into_element().into_any()
    }
}
```

<Info>
Many types implement `IntoElement`, including `SharedString`, `String`, and types that derive `IntoElement`.
</Info>

## Common Elements

GPUI provides several built-in element types.

### Div Element

The `div` element is an all-purpose container element.

```rust Using div
use gpui::{div, prelude::*, px, rgb};

div()
    .id("container")
    .flex()
    .flex_col()
    .gap_4()
    .p_4()
    .bg(rgb(0xffffff))
    .rounded_lg()
    .border_1()
    .border_color(rgb(0xe5e7eb))
    .child("First child")
    .child(div().child("Nested child"))
```

### Stateful Elements

Stateful elements have IDs and can maintain state across frames.

```rust Stateful div
use gpui::{Stateful, Div, ElementId};

let stateful_div: Stateful<Div> = div()
    .id("my-element")
    .bg(gpui::blue());
```

## Element State

Elements can maintain state across frames using `RequestLayoutState` and `PrepaintState`.

<Tabs>
  <Tab title="RequestLayoutState">
    State returned from `request_layout()`. Used for layout calculations.
    
    ```rust
    type RequestLayoutState = MyLayoutState;
    
    struct MyLayoutState {
        child_layout_id: LayoutId,
    }
    ```
  </Tab>
  <Tab title="PrepaintState">
    State returned from `prepaint()`. Used for hit testing and painting.
    
    ```rust
    type PrepaintState = MyPrepaintState;
    
    struct MyPrepaintState {
        hovered: bool,
        bounds: Bounds<Pixels>,
    }
    ```
  </Tab>
</Tabs>

## Element IDs

Elements can have unique identifiers for tracking across frames.

```rust Element IDs
use gpui::ElementId;

// String-based ID
div().id("my-element")

// Integer-based ID
div().id(42)

// Custom ID type
let custom_id = ElementId::Name("custom".into());
div().id(custom_id)
```

<Info>
The global ID is used to access state connected to an element with the same ID across frames. This ID must be unique among children of the first containing element with an ID.
</Info>

## Layout with Flexbox

GPUI uses Taffy for flexbox layout, similar to CSS.

<CodeGroup>

```rust Flex container
div()
    .flex()
    .flex_row()        // flex-direction: row
    .justify_between() // justify-content: space-between
    .items_center()    // align-items: center
    .gap_4()           // gap: 1rem
    .child(div().flex_1().child("Grows"))
    .child(div().child("Fixed"))
```

```rust Flex column
div()
    .flex()
    .flex_col()        // flex-direction: column
    .justify_start()   // justify-content: flex-start
    .items_stretch()   // align-items: stretch
    .h_full()          // height: 100%
    .child(div().h(px(100.)).child("Fixed height"))
    .child(div().flex_1().child("Fills remaining"))
```

```rust Flex wrap
div()
    .flex()
    .flex_wrap()       // flex-wrap: wrap
    .gap_2()
    .child(div().w(px(100.)).child("Item 1"))
    .child(div().w(px(100.)).child("Item 2"))
    .child(div().w(px(100.)).child("Item 3"))
```

</CodeGroup>

## Interactive States

Elements can respond to hover, focus, and other states.

<CodeGroup>

```rust Hover state
div()
    .bg(gpui::white())
    .hover(|style| {
        style.bg(gpui::gray())
    })
    .child("Hover me")
```

```rust Focus state
div()
    .id("focusable")
    .track_focus(&focus_handle)
    .focus(|style| {
        style.border_2().border_color(gpui::blue())
    })
    .child("Focusable element")
```

```rust Focus visible
div()
    .track_focus(&focus_handle)
    .focus_visible(|style| {
        style.border_2().border_color(gpui::green())
    })
    .child("Shows border only with keyboard focus")
```

</CodeGroup>

## Custom Elements

<Warning>
Implement custom elements only when you need manual control of layout and painting, such as for custom layout algorithms or specialized rendering like code editors.
</Warning>

```rust Custom element example
use gpui::*;

struct CustomElement {
    content: String,
}

impl Element for CustomElement {
    type RequestLayoutState = ();
    type PrepaintState = ();

    fn id(&self) -> Option<ElementId> {
        None
    }
    
    fn source_location(&self) -> Option<&'static panic::Location<'static>> {
        None
    }

    fn request_layout(
        &mut self,
        _id: Option<&GlobalElementId>,
        _inspector_id: Option<&InspectorElementId>,
        window: &mut Window,
        cx: &mut App,
    ) -> (LayoutId, Self::RequestLayoutState) {
        let style = Style::default();
        let layout_id = window.request_layout(style, None, cx);
        (layout_id, ())
    }

    fn prepaint(
        &mut self,
        _id: Option<&GlobalElementId>,
        _inspector_id: Option<&InspectorElementId>,
        _bounds: Bounds<Pixels>,
        _request_layout: &mut Self::RequestLayoutState,
        _window: &mut Window,
        _cx: &mut App,
    ) -> Self::PrepaintState {
        ()
    }

    fn paint(
        &mut self,
        _id: Option<&GlobalElementId>,
        _inspector_id: Option<&InspectorElementId>,
        bounds: Bounds<Pixels>,
        _request_layout: &mut Self::RequestLayoutState,
        _prepaint: &mut Self::PrepaintState,
        window: &mut Window,
        _cx: &mut App,
    ) {
        // Custom painting logic here
    }
}
```

## Content Masking

Elements can ignore parent bounds and paint anywhere inside the window.

```rust Content mask
window.with_content_mask(content_mask, |window| {
    // Elements here can paint outside normal bounds
    // Useful for overlays and popups
});
```
