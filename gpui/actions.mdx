## Overview

GPUI is designed for keyboard-first interactivity. Actions are used to implement keyboard-driven UI by binding keys to actions in the keymap and registering listeners for those actions in the element tree.

<Note>
To expose functionality to the mouse, render a button with a click handler. To expose functionality to the keyboard, bind an action in a key context.
</Note>

## What are Actions?

Actions are similar to framework-level events like `MouseDown` and `KeyDown`, but you define them yourself.

```rust Simple actions
mod menu {
    use gpui::actions;
    
    actions!(gpui, [MoveUp, MoveDown]);
}
```

<Info>
Actions are unit structs created with the `actions!` macro for simple cases.
</Info>

## Defining Actions

### Using the actions! Macro

For simple unit struct actions:

```rust actions! macro
use gpui::actions;

actions!(editor, [MoveUp, MoveDown, MoveLeft, MoveRight, Newline]);
```

This creates actions with names like `editor::MoveUp`, `editor::MoveDown`, etc.

### Using the Action Derive Macro

For complex data types:

<CodeGroup>

```rust Simple action
use gpui::Action;

#[derive(Clone, PartialEq, Action)]
#[action(namespace = editor)]
pub struct SelectNext {
    pub replace_newest: bool,
}
```

```rust Action with direction
#[derive(Clone, PartialEq, Action)]
#[action(namespace = menu)]
pub struct Move {
    direction: Direction,
    select: bool,
}
```

</CodeGroup>

<Warning>
The `Action` derive macro requires that the type implement `Clone` and `PartialEq`. It also requires `serde::Deserialize` and `schemars::JsonSchema` unless `#[action(no_json)]` is specified.
</Warning>

### Action Attributes

<AccordionGroup>
  <Accordion title="namespace">
    Sets the action's namespace:
    
    ```rust
    #[action(namespace = editor)]
    pub struct MoveUp;
    ```
  </Accordion>
  
  <Accordion title="name">
    Overrides the action's name:
    
    ```rust
    #[action(name = "CustomName")]
    pub struct MyAction;
    ```
    
    The name must not contain `::`.
  </Accordion>
  
  <Accordion title="no_json">
    Disables JSON deserialization:
    
    ```rust
    #[action(no_json)]
    pub struct InternalAction;
    ```
    
    Allows actions to skip `serde::Serialize` and `schemars::JsonSchema`.
  </Accordion>
  
  <Accordion title="no_register">
    Skips action registration:
    
    ```rust
    #[action(no_register)]
    pub struct UnregisteredAction;
    ```
  </Accordion>
  
  <Accordion title="deprecated_aliases">
    Specifies deprecated old names:
    
    ```rust
    #[action(deprecated_aliases = ["editor::OldName"])]
    pub struct NewAction;
    ```
  </Accordion>
  
  <Accordion title="deprecated">
    Marks action as deprecated:
    
    ```rust
    #[action(deprecated = "Use NewAction instead")]
    pub struct OldAction;
    ```
  </Accordion>
</AccordionGroup>

## Binding Actions to Elements

Use `.on_action()` to register action handlers on elements.

```rust Binding actions
use gpui::{actions, div, prelude::*, Context, Render, Window};

actions!(menu, [MoveUp, MoveDown]);

struct Menu;

impl Render for Menu {
    fn render(&mut self, window: &mut Window, cx: &mut Context<Self>) -> impl IntoElement {
        div()
            .on_action(|this: &mut Menu, action: &MoveUp, window: &mut Window, cx: &mut Context<Menu>| {
                // Handle move up
            })
            .on_action(|this, action: &MoveDown, window, cx| {
                // Handle move down
            })
            .child("Menu")
    }
}
```

<Info>
Like other event handlers, `.on_action()` is often used with `cx.listener()` to automatically receive `this: &mut T`.
</Info>

### Using cx.listener

```rust Using cx.listener
impl Render for Menu {
    fn render(&mut self, window: &mut Window, cx: &mut Context<Self>) -> impl IntoElement {
        div()
            .on_action(cx.listener(|this: &mut Menu, action: &MoveUp, window, cx| {
                this.selected_index = this.selected_index.saturating_sub(1);
                cx.notify();
            }))
            .on_action(cx.listener(|this, action: &MoveDown, window, cx| {
                this.selected_index = (this.selected_index + 1).min(this.items.len() - 1);
                cx.notify();
            }))
            .child("Menu")
    }
}
```

## Key Contexts

To bind keys to actions, declare a _key context_ using `.key_context()`.

```rust Key context
impl Render for Menu {
    fn render(&mut self, window: &mut Window, cx: &mut Context<Self>) -> impl IntoElement {
        div()
            .key_context("menu")
            .on_action(cx.listener(|this, _: &MoveUp, window, cx| {
                // ...
            }))
            .on_action(cx.listener(|this, _: &MoveDown, window, cx| {
                // ...
            }))
            .child("Menu")
    }
}
```

## Keymap Configuration

Bind keys to actions in the keymap using JSON.

<Tabs>
  <Tab title="Simple actions">
    ```json
    {
      "context": "menu",
      "bindings": {
        "up": "menu::MoveUp",
        "down": "menu::MoveDown"
      }
    }
    ```
    
    Actions are identified by their fully-qualified type name.
  </Tab>
  
  <Tab title="Actions with data">
    ```json
    {
      "context": "menu",
      "bindings": {
        "up": ["menu::Move", {"direction": "up", "select": false}],
        "down": ["menu::Move", {"direction": "down", "select": false}],
        "shift-up": ["menu::Move", {"direction": "up", "select": true}],
        "shift-down": ["menu::Move", {"direction": "down", "select": true}]
      }
    }
    ```
    
    Provide the serialized representation of the action alongside the name.
  </Tab>
</Tabs>

## Binding Keys in Code

You can also bind keys programmatically.

```rust Programmatic key bindings
use gpui::{App, KeyBinding};

actions!(example, [Tab, TabPrev, Quit]);

application().run(|cx: &mut App| {
    cx.bind_keys([
        KeyBinding::new("tab", Tab, None),
        KeyBinding::new("shift-tab", TabPrev, None),
        KeyBinding::new("cmd-q", Quit, None),
    ]);
});
```

## Dispatching Actions

Actions can be dispatched programmatically.

<CodeGroup>

```rust Via window
use gpui::Action;

window.dispatch_action(SomeAction.boxed_clone(), cx);
```

```rust Via focus handle
focus_handle.dispatch_action(&SomeAction, window, cx);
```

</CodeGroup>

## Complete Example

Here's a complete example with actions, key bindings, and event handlers.

```rust Complete action example
use gpui::{
    actions, div, prelude::*, App, Bounds, Context, FocusHandle, KeyBinding,
    Render, SharedString, Window, WindowBounds, WindowOptions, px, size,
};

actions!(example, [Tab, TabPrev, Quit]);

struct Example {
    focus_handle: FocusHandle,
    items: Vec<(FocusHandle, &'static str)>,
    message: SharedString,
}

impl Example {
    fn new(window: &mut Window, cx: &mut Context<Self>) -> Self {
        let items = vec![
            (
                cx.focus_handle().tab_index(1).tab_stop(true),
                "Button 1",
            ),
            (
                cx.focus_handle().tab_index(2).tab_stop(true),
                "Button 2",
            ),
        ];

        let focus_handle = cx.focus_handle();
        window.focus(&focus_handle, cx);

        Self {
            focus_handle,
            items,
            message: "Press Tab to navigate!".into(),
        }
    }

    fn on_tab(&mut self, _: &Tab, window: &mut Window, cx: &mut Context<Self>) {
        window.focus_next(cx);
        self.message = "Pressed Tab".into();
        cx.notify();
    }

    fn on_tab_prev(&mut self, _: &TabPrev, window: &mut Window, cx: &mut Context<Self>) {
        window.focus_prev(cx);
        self.message = "Pressed Shift-Tab".into();
        cx.notify();
    }

    fn on_quit(&mut self, _: &Quit, _window: &mut Window, cx: &mut Context<Self>) {
        cx.quit();
    }
}

impl Render for Example {
    fn render(&mut self, _window: &mut Window, cx: &mut Context<Self>) -> impl IntoElement {
        div()
            .id("app")
            .track_focus(&self.focus_handle)
            .on_action(cx.listener(Self::on_tab))
            .on_action(cx.listener(Self::on_tab_prev))
            .on_action(cx.listener(Self::on_quit))
            .size_full()
            .flex()
            .flex_col()
            .gap_4()
            .child(self.message.clone())
            .children(self.items.iter().enumerate().map(|(i, (handle, label))| {
                div()
                    .id(format!("button-{}", i))
                    .track_focus(handle)
                    .p_4()
                    .bg(gpui::blue())
                    .text_color(gpui::white())
                    .rounded_md()
                    .focus(|style| style.border_2().border_color(gpui::yellow()))
                    .child(*label)
            }))
    }
}

fn main() {
    gpui_platform::application().run(|cx: &mut App| {
        cx.bind_keys([
            KeyBinding::new("tab", Tab, None),
            KeyBinding::new("shift-tab", TabPrev, None),
            KeyBinding::new("cmd-q", Quit, None),
        ]);

        let bounds = Bounds::centered(None, size(px(600.), px(400.0)), cx);
        cx.open_window(
            WindowOptions {
                window_bounds: Some(WindowBounds::Windowed(bounds)),
                ..Default::default()
            },
            |window, cx| cx.new(|cx| Example::new(window, cx)),
        )
        .unwrap();

        cx.activate(true);
    });
}
```

## Action Best Practices

<Steps>
  <Step title="Namespace actions">
    Always use namespaces to avoid conflicts: `actions!(editor, [...])`
  </Step>
  <Step title="Document actions">
    Doc comments on actions are displayed to users
  </Step>
  <Step title="Use key contexts">
    Declare key contexts to scope keybindings appropriately
  </Step>
  <Step title="Handle state updates">
    Call `cx.notify()` after modifying state in action handlers
  </Step>
</Steps>

<Warning>
Registering actions with the same name will result in a panic during `App` creation.
</Warning>

## Action Dispatch Flow

<Steps>
  <Step title="User input">
    User presses a key combination
  </Step>
  <Step title="Keymap lookup">
    GPUI looks up the action in the keymap for the current key context
  </Step>
  <Step title="Action dispatch">
    The action is dispatched to the focused element
  </Step>
  <Step title="Handler execution">
    The action handler is called with the action data
  </Step>
  <Step title="Bubbling">
    If not handled, the action bubbles up the element tree
  </Step>
</Steps>
