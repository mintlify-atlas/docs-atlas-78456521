## Bird's-eye view of Zed

Zed is built as a modular system of Rust crates, each handling specific aspects of the editor's functionality. Understanding this architecture is essential for contributing effectively to the project.

<Tip>
Keep the [Zed glossary](/development/glossary) at your side when starting out. It lists and explains key structures and terms you'll encounter throughout the codebase.
</Tip>

## Core crates

Zed is made up of several smaller crates. Here are the ones you're most likely to interact with:

### gpui

A GPU-accelerated UI framework providing all building blocks for Zed. GPUI handles state management, rendering, and event handling.

<Warning>
Familiarize yourself with the root level GPUI documentation before making UI changes. Understanding GPUI's patterns is crucial for effective development.
</Warning>

Key concepts:
- `App` - Root context for global state and entity management
- `Entity<T>` - Handles to managed state
- `Window` - Window state and rendering
- `Context<T>` - Entity-specific context for updates

### editor

Contains the core `Editor` type driving both the code editor and input fields throughout Zed. Also handles the display layer for LSP features like inlay hints and code completions.

### project

Manages files and navigation within the file tree. Handles Zed's communication with Language Server Protocol (LSP) servers.

### workspace

Handles local state serialization and groups projects together.

### vim

A thin implementation of Vim workflow layered over the `editor` crate.

### lsp

Handles communication with external LSP servers.

### language

Drives the editor's understanding of programming languages, from symbol lists to syntax maps.

### collab

The collaboration server driving features like project sharing.

### rpc

Defines messages exchanged with the collaboration server.

### theme

Defines the theme system and provides default themes.

### ui

A collection of UI components and common patterns used throughout Zed.

### cli

The CLI crate which invokes the Zed binary.

### zed

Where all components come together. Contains the `main` entry point for Zed.
